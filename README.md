# FALIZ Chat Bot - Multilingual AI Assistant

A production-ready, multilingual Flask-based chat application with real-time language detection and OpenAI integration.

## 🌟 Features

- **🌍 Multilingual Support**: English, Spanish, French, Portuguese
- **🔍 Real-time Language Detection**: Automatic detection using `langdetect`
- **💬 Code-Switching Support**: Bot mirrors language mixing in user messages
- **💾 Persistent Conversations**: SQLite database stores all conversations
- **🎨 Beautiful Dark UI**: Modern, responsive web interface with message bubbles
- **📱 Mobile Responsive**: Works on all devices
- **🚀 Easy Deployment**: Replit, PythonAnywhere, Railway, and more
- **🔄 Fallback Support**: Graceful degradation when API fails

## 📋 Prerequisites

- Python 3.8+
- OpenAI API Key (get it from [openai.com/api/](https://platform.openai.com/api/keys))
- Git (for cloning)

## 🛠️ Installation

### Local Setup

1. **Clone the repository**
```bash
git clone https://github.com/FALIZ-Tech/WEB-HOOK-2.git
cd WEB-HOOK-2
```

2. **Create virtual environment**
```bash
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
```

3. **Install dependencies**
```bash
pip install -r requirements.txt
```

4. **Set up environment variables**
```bash
cp .env.example .env
# Edit .env and add your OpenAI API key
nano .env  # or use your preferred editor
```

5. **Run the application**
```bash
python app.py
```

6. **Access the app**
- Open `http://localhost:5000` in your browser

## 🔄 Fallback Chat Flow

When the OpenAI API fails, the application automatically activates a **fallback mode** that provides intelligent responses without requiring the API:

### How It Works

1. **Intelligent Pattern Matching**
   - Detects keywords and phrases in user messages
   - Provides contextual responses based on language

2. **Multi-Language Support**
   - Generates responses in the user's detected language
   - English, Spanish, French, Portuguese support

3. **Conversation Context**
   - References previous messages
   - Maintains conversation flow

### Fallback Responses Include

- **Greetings**: "Hello", "Hi", "Hola", "Bonjour"
- **Questions**: "How are you?", "What's your name?"
- **Help Requests**: "Can you help me?", "I need assistance"
- **Goodbyes**: "Bye", "See you", "Adiós"
- **General Chat**: Intelligent fallback responses

### Triggering Fallback

Fallback automatically activates when:
- OpenAI API key is missing
- API returns an error
- Network connection fails
- API rate limit exceeded

**Users see a subtle indicator** that fallback mode is active but continue getting responses.

### Example Fallback Exchange

```
User (Spanish): "¿Cómo estás?"
Bot (Fallback): "¡Hola! Estoy aquí para ayudarte. 
Aunque no pueda conectarme a mi IA avanzada en este momento, 
seguiré haciendo mi mejor esfuerzo para asistirte. 
¿En qué puedo ayudarte?"

User: "Can you help me with coding?"
Bot (Fallback): "I'd be happy to help with coding! 
However, I'm currently operating in fallback mode. 
For more advanced assistance, please try again in a moment. 
What programming language are you working with?"
```

## 🐛 Troubleshooting

### Issue: "ModuleNotFoundError"
**Solution:**
```bash
pip install -r requirements.txt
```

### Issue: "OPENAI_API_KEY not found"
**Solution:**
1. Create `.env` file in project root
2. Add: `OPENAI_API_KEY=sk-...`
3. Restart the application

### Issue: "Connection refused"
**Solution:**
- Make sure Flask app is running
- Check if port 5000 is available
- Try different port: `python app.py --port 8000`

### Issue: "Bot not responding"
**Solution:**
1. Check API key validity
2. Verify internet connection
3. Check OpenAI account credits
4. Application will use fallback if API fails

### Issue: "Language not detected"
**Solution:**
- Try with longer messages (at least 3-4 words)
- Use correct spelling and grammar
- The app defaults to English if detection fails

## 📊 Database Structure

### Conversations Table
```sql
CREATE TABLE conversations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
)
```

### Messages Table
```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL,
    role TEXT NOT NULL,
    content TEXT NOT NULL,
    detected_language TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY(conversation_id) REFERENCES conversations(id)
)
```
